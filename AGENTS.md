# AGENTS.md

## Overview

This is a **Quartz v4** site — a static site generator for digital gardens. It transforms markdown files in `content/` into a searchable, browsable website.

## Key commands

- `npm run quartz build` — full production build
- `npm run docs build --serve` — dev server with live reload (default port 3009)
- `npm run check` — run `tsc --noEmit && npx prettier . --check`
- `npm run format` — auto-format all files with prettier
- `npm run test` — run tsx test suite
- `npm run profile` — profile build performance with 0x

## Architecture

- **`quartz/`** — framework source code (plugins, components, processors, i18n)
- **`content/`** — user's Obsidian vault (markdown content). Do not edit `content/.obsidian/`
- **`quartz/bootstrap-cli.mjs`** — the CLI entry point (not TypeScript)
- **`quartz/util/`** — path, emoji, theme, log, perf utilities
- **`quartz/plugins/`** — transformers (markdown processing), emitters (HTML output), filters (routing)
- **`quartz/components/`** — Preact components (search, graph, explorer, dark mode, etc.)

## Tech constraints

- **Node >= 22** required (check `package.json` engines)
- TypeScript strict mode (`noUnusedLocals`, `noUnusedParameters`)
- JSX via Preact (`jsxImportSource: preact`)
- esbuild for bundling (not webpack)
- `tsx` for running tests (not jest/vitest)

## Content conventions

- Frontmatter fields: `title`, `tags`, `date`, `draft`, `description`
- Aliases via explicit `aliases` frontmatter or `content/aliases/` directory
- `tags` are hierarchical (`platform-engineering/kubernetes`)

## Build pipeline

Parse → Filter → Transform → Emit. See `quartz/build.ts` and `quartz/cfg.ts` for the full pipeline.

## Gotchas

- `quartz/bootstrap-cli.mjs` is plain JS/ESM — don't add TypeScript annotations there
- `.obsidian/` workspace files are auto-generated by Obsidian — don't edit them manually
- `quartz/util/emojimap.json` is auto-generated; don't manually edit
